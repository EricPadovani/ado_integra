Visão Geral

O ADO Iterations Manager é uma aplicação Node.js que integra Azure DevOps, SAP RFC e Inteligência Artificial (Claude API) para fornecer uma visão consolidada das iterações (sprints) do projeto Mastersaf Interfaces.

O sistema consulta a API REST do Azure DevOps, gera um relatório HTML interativo e permite, a partir do browser, visualizar os Work Items de cada sprint, analisar automaticamente processos SAP vinculados e obter uma análise de cobertura gerada por IA.

Ponto de entrada: busca_ado.js — gera o arquivo iteracoes_ado.html que é aberto diretamente no navegador. Toda a interatividade posterior (chamadas ADO, SAP, IA) ocorre no frontend via fetch.

Configuração e Instalação

Pré-requisitos

  • Node.js 18 ou superior
  • Acesso à organização Azure DevOps tr-ggo
  • Token PAT do Azure DevOps com permissão de leitura em Work Items
  • Servidor api-server.js em execução na porta 3001 (para Análise IA)

Instalação de Dependências

cd C:/vscode/ado
npm install

Variável de Ambiente

Criar arquivo .env na raiz do projeto:

ADO_TOKEN=<seu-token-PAT-em-base64>
O token deve ser codificado em Base64 no formato :<PAT> (dois-pontos antes do PAT), como exige a autenticação Basic do Azure DevOps.

Execução

# Gerar o relatório HTML
node busca_ado.js

# Iniciar o servidor de API (para Análise IA)
node api-server.js   # ou via iniciar.bat

Funcionalidades — Tela Principal

Ao executar node busca_ado.js, o arquivo iteracoes_ado.html é gerado e pode ser aberto no navegador.

Header com Estatísticas

O cabeçalho exibe o nome do projeto e badges com contagens:

  • Total — número total de iterações encontradas
  • Concluídas — iterações com timeFrame = past
  • Atual — iteração com timeFrame = current
  • Futuras — iterações com timeFrame = future

Filtros de Status

Botões de filtro rápido na barra superior da tabela:

BotãoComportamentoPadrão ativo?
TodasExibe todas as iteraçõesNão
ConcluídasExibe apenas iterações passadas (past)Não
AtuaisExibe apenas a iteração corrente (current)Sim
FuturasExibe apenas iterações futuras (future)Não

Tabela de Iterações

Colunas exibidas:

  • # — número sequencial
  • Nome da Iteração — nome do sprint
  • Caminho — path no ADO, com prefixo Mastersaf Interfaces\ removido; clicável para abrir o modal de Work Items
  • Data Início / Data Fim — formatadas no padrão dd/mm/aaaa

  • Link — link direto para a iteração no portal Azure DevOps

Rodapé

Exibe data/hora de geração do arquivo e a origem dos dados (Azure DevOps API).

Funcionalidades — Modal Work Items

Ao clicar no Caminho de uma iteração, um modal é aberto com os Work Items daquela sprint. Apenas itens do tipo User Story são exibidos.

Fluxo de Carregamento

  1. Requisição à API ADO: iterations/{id}/workitems para obter a lista de IDs
  2. Para cada ID: requisição a /wit/workitems/{id} para buscar detalhes
  3. Filtro client-side: apenas System.WorkItemType === 'User Story'
  4. Exibição em tabela com contagem de User Stories encontradas

Colunas da Tabela de Work Items

ColunaDescrição
IDID numérico clicável (exibe alert com ID e título — ponto de extensão para futuras integrações)
TipoTipo do Work Item (sempre User Story nesta view)
TítuloTítulo da User Story
EstadoEstado atual no ADO (Active, Resolved, Closed, etc.)
LinkLink direto para o item no portal Azure DevOps
Análise IABotão 🤖 ANÁLISE IA para abrir modal de análise
Em caso de erro ao carregar um work item específico, a linha exibe "Não foi possível carregar" e um link direto para o Azure, garantindo que o modal não trave.

Funcionalidades — Análise IA

O modal de Análise IA realiza uma análise automatizada cruzando a especificação da User Story (ADO) com o código-fonte SAP (via RFC) e gera um parecer com IA.

Etapas do Fluxo

  1. Busca detalhes ADO — campos System.Title, System.State, System.Description, System.AssignedTo, System.IterationPath
  2. Extrai código ZSAFE dos comentários — busca comentários do Work Item e extrai padrões Processo: ZSAFE... via regex /Processo:\s*(\S+)/gi
  3. Consulta RFC SAP — chama localhost:3001/api/analise com o workItemId e o código ZSAFE; o servidor executa a RFC Z_IN_PGM
  4. Análise com IA — envia spec ADO + código SAP para localhost:3001/api/ia-analise, que retorna análise formatada em Markdown
  5. Renderização — a função formatarAnaliseIA() converte o Markdown (## títulos, listas) para HTML formatado

Caso de ZSAFE não encontrado

Se nenhum padrão Processo: ZSAFE... for encontrado nos comentários, o modal exibe um formulário para que o usuário informe manualmente o código. A função reConsultarZsafe() reexecuta o fluxo com o código informado.

Suporte a múltiplos ZSAFEs

Se mais de um código for detectado nos comentários, o sistema itera sobre todos, exibindo a análise SAP + IA de cada um em sequência.

Resultado exibido

SeçãoOrigemEstilo
🔵 Azure DevOpsADO APICard azul com tabela de detalhes
❌ Erros SAP/IAServidor localCard vermelho com mensagem
🤖 Análise IAClaude API via servidorCard cinza-azulado com conteúdo Markdown renderizado

Funcionalidade — Painel Pontos Ajustados

Status: Implementação planejada (plano em docs/superpowers/plans/2026-03-31-pontos-ajustados-panel.md)

Quando a RFC Z_IN_PGM retornar números de Work Items ADO nos campos O_FONTEFF, O_FONTEFS ou O_FONTEFM, o modal de Análise IA exibirá um painel lateral direito com cards dos work items referenciados.

Layout 2 Colunas

O modal passa a ter estrutura de 2 colunas:

  • Coluna esquerda (55%) — análise existente (SAP + IA + detalhes ADO)
  • Coluna direita (42%) — cards dos Work Items referenciados nos campos RFC

Cards de Work Item Referenciado

Cada card exibe:

  • Campo de origem (O_FONTEFF, O_FONTEFS ou O_FONTEFM) e ID do Work Item
  • Título, Tipo, Estado, Responsável, Sprint
  • Link direto para o Azure DevOps

Comportamento adaptativo

  • Se todos os campos estiverem vazios ou zerados: coluna direita é ocultada (display:none), mantendo layout original de 1 coluna
  • Em telas menores (< 768px): layout empilhado verticalmente via media query

Função responsável

renderPontosAjustados(container, sapData, adoToken) — função assíncrona que recebe os dados SAP parseados, detecta os campos, busca os Work Items na ADO API e renderiza os cards no container da coluna direita.

API Endpoints (api-server.js)

O servidor Express local (localhost:3001) atua como proxy para evitar problemas de CORS e para manter as credenciais SAP/Claude fora do browser.

Endpoint Método Parâmetros Descrição
/api/analise GET workItemId, iZsafe Executa RFC Z_IN_PGM no SAP e retorna o código-fonte do processo
/api/ia-analise POST JSON: adoSpec, sapCode, zsafe, workItemId Envia especificação + código para Claude API e retorna análise em Markdown

Resposta de sucesso

// GET /api/analise
{ "success": true, "data": { /* objeto RFC */ } }

// POST /api/ia-analise
{ "success": true, "analysis": "## Cobertura\n- ..." }

Resposta de erro

{ "success": false, "error": "mensagem de erro" }
O servidor deve estar em execução antes de usar o botão "🤖 ANÁLISE IA" no browser. Execute com node api-server.js ou use o atalho iniciar.bat.

Fluxo de Dados

Geração do Relatório (busca_ado.js)

1. Lê ADO_TOKEN do .env 2. GET /work/teamsettings/iterations?api-version=7.0 3. Recebe lista de iterações com: id, name, path, startDate, finishDate, timeFrame 4. gerarHTML(dados) → template HTML completo com CSS e JS inline 5. salvarHTML(html, 'iteracoes_ado.html') → grava no disco 6. exibirTabela(dados) → exibe console.table no terminal 7. selecionarIteracao(dados) → aguarda input do usuário (extensível)

Análise IA (browser → api-server)

Browser: clique em 🤖 ANÁLISE IA │ ├─ GET ADO /wit/workitems/{id}?$expand=all │ └─ extrai: title, state, description, assignedTo, iteration │ ├─ GET ADO /wit/workitems/{id}/comments │ └─ regex: Processo: ZSAFE... → zsafeCodes[] │ ├─ [se zsafeCodes.length === 0] → exibe formulário manual │ └─ para cada zsafe em zsafeCodes: ├─ GET localhost:3001/api/analise?workItemId=X&iZsafe=ZSAFE │ └─ SAP RFC Z_IN_PGM → retorna sapCode (JSON) │ ├─ POST localhost:3001/api/ia-analise │ body: { adoSpec, sapCode, zsafe, workItemId } │ └─ Claude API → análise Markdown │ └─ renderiza: card SAP + card IA + card ADO + [se O_FONTEFF/S/M preenchidos] → renderPontosAjustados()

Variáveis de Ambiente

Variável Onde usada Formato Obrigatória?
ADO_TOKEN busca_ado.js (geração) e embutida no HTML para chamadas browser Base64 de :<PAT> Sim
Atenção de segurança: O ADO_TOKEN é embutido diretamente no HTML gerado para permitir as chamadas de API do browser. O arquivo iteracoes_ado.html deve ser tratado como documento sensível e não deve ser publicado ou compartilhado em repositórios públicos.

Variáveis no api-server.js

O servidor de API gerencia separadamente as credenciais SAP e a chave da Claude API. Consulte o arquivo api-server.js ou seu próprio .env para as variáveis específicas daquele serviço.
